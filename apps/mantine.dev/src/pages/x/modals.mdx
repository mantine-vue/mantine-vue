## Setup ModalsProvider

Wrap your app with the `ModalsProvider` component:

```vue
<script setup lang="ts">
import { ModalsProvider } from '@mantine-vue/modals'
</script>

<template>
  <ModalsProvider>
    <!-- Your app here -->
  </ModalsProvider>
</template>
```

## Confirm modal

The `@mantine-vue/modals` package includes a special modal that can be used for confirmations.
The component includes confirm and cancel buttons and supports children to display additional
information about the action. Use the `modals.openConfirmModal` function to open a confirm modal:

<Demo data={ModalsDemos.confirm} />

`modals.openConfirmModal` accepts one argument with the following properties:

- `modalId` â€“ modal id, defaults to a random id, can be used to close the modal programmatically
- `children` â€“ additional modal content displayed before actions (a VNode, array of VNodes, or a function that returns one)
- `onCancel` â€“ called when the cancel button is clicked
- `onConfirm` â€“ called when the confirm button is clicked
- `closeOnConfirm` â€“ should the modal be closed when the confirm button is clicked, defaults to `true`
- `closeOnCancel` â€“ should the modal be closed when the cancel button is clicked, defaults to `true`
- `cancelProps` â€“ cancel button props
- `confirmProps` â€“ confirm button props
- `groupProps` â€“ buttons [Group](/core/group) props
- `labels` â€“ cancel and confirm button labels, can be defined on `ModalsProvider`

Using these properties, you can customize the confirm modal to match current context requirements:

<Demo data={ModalsDemos.confirmCustomize} />

To set up shared labels for confirm modals, set `labels` on `ModalsProvider`:

```vue
<script setup lang="ts">
import { ModalsProvider } from '@mantine-vue/modals'
</script>

<template>
  <ModalsProvider :labels="{ confirm: 'Submit', cancel: 'Cancel' }">
    <!-- Your app here -->
  </ModalsProvider>
</template>
```

## Context modals

You can define any number of modals in the `ModalsProvider` context by passing a `modals` prop â€“
a record of component names to components. Every context modal component receives `context`,
`id` and `innerProps` props:

```vue
<script setup lang="ts">
import { Button, Text } from '@mantine-vue/core'
import { ModalsProvider } from '@mantine-vue/modals'
import type { ContextModalProps } from '@mantine-vue/modals'
import { defineComponent, h } from 'vue'

const TestModal = defineComponent({
  props: ['context', 'id', 'innerProps'],
  setup(props: ContextModalProps<{ modalBody: string }>) {
    return () => [
      h(Text, { size: 'sm' }, () => props.innerProps.modalBody),
      h(
        Button,
        { fullWidth: true, mt: 'md', onClick: () => props.context.closeModal(props.id) },
        () => 'Close modal',
      ),
    ]
  },
})
</script>

<template>
  <ModalsProvider :modals="{ demonstration: TestModal }">
    <!-- Your app here -->
  </ModalsProvider>
</template>
```

And then open one of these modals with the `modals.openContextModal` function. It accepts a single
payload object with a `modal` key that matches one of the keys defined on `ModalsProvider`:

## useModals composable

`useModals` returns the same context object that context modal components receive as their
`context` prop â€“ it gives access to `modals`, `modalProps`, and all the `open`/`close`/`update`
methods from anywhere inside the `ModalsProvider` tree:

```vue
<script setup lang="ts">
import { useModals } from '@mantine-vue/modals'

const ctx = useModals()
</script>
```

`useModals` throws an error when called outside of `ModalsProvider`, matching the behavior of the
standalone `modals` object, which relies on the default store created by `ModalsProvider` always
being mounted somewhere in the app.
