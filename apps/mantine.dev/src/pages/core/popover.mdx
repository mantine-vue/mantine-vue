## Controlled

You can control the Popover state with the `opened` and `@change` props:

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { Button, Popover } from '@mantine-vue/core'

const opened = ref(false)
</script>

<template>
  <Popover :opened="opened" @change="opened = $event">
    <Popover.Target>
      <Button @click="opened = !opened">Toggle popover</Button>
    </Popover.Target>
    <Popover.Dropdown>Dropdown</Popover.Dropdown>
  </Popover>
</template>
```

## Click outside

By default, `Popover` closes when you click outside of the dropdown. To disable this behavior, set `:close-on-click-outside="false"`.

You can configure events used for click-outside detection with `clickOutsideEvents`:

## onDismiss

If you need to control the opened state, but still want to close the popover on outside clicks
and Escape key presses, use the `@dismiss` event:

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { Button, Popover } from '@mantine-vue/core'

const opened = ref(false)
</script>

<template>
  <Popover :opened="opened" @dismiss="opened = false">
    <Popover.Target>
      <Button @click="opened = !opened">Toggle popover</Button>
    </Popover.Target>
    <Popover.Dropdown>Dropdown</Popover.Dropdown>
  </Popover>
</template>
```

## Initial focus

Popover uses the [FocusTrap](/core/focus-trap/) component to manage focus.
Add the `data-autofocus` attribute to the element that should receive initial focus:

```vue
<template>
  <Popover>
    <Popover.Target>
      <button type="button">Target</button>
    </Popover.Target>
    <Popover.Dropdown>
      <input />
      <input data-autofocus />
      <input />
    </Popover.Dropdown>
  </Popover>
</template>
```

## Accessibility

Popover follows [WAI-ARIA recommendations](https://www.w3.org/TR/wai-aria-practices-1.2/#dialog_modal):

- Dropdown element has `role="dialog"` and `aria-labelledby` attributes
- Target element has `aria-haspopup="dialog"`, `aria-expanded`, and `aria-controls` attributes
