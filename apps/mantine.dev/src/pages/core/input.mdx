## Disclaimer

**!important:** In most cases, you should not use `Input` in your application.
`Input` is a base for other inputs and was not designed to be used directly.
Use `Input` to create custom inputs. For other cases, prefer [TextInput](/core/text-input/)
or another component.

```vue
<script setup lang="ts">
import { Input, InputWrapper, TextInput } from '@mantine-vue/core'
</script>

<template>
  <!-- Incorrect usage, input is not accessible -->
  <InputWrapper label="Input label">
    <Input />
  </InputWrapper>

  <!-- Use TextInput instead – it is accessible by default and includes InputWrapper -->
  <TextInput label="Input label" description="Input description" />
</template>
```

## Usage

The `Input` component is used as a base for some other inputs ([NativeSelect](/core/native-select/), [TextInput](/core/text-input/), [Textarea](/core/textarea/), etc.).
The purpose of the `Input` is to provide shared styles and features to other inputs.

## InputWrapper component

The `InputWrapper` component is used in all other inputs
([TextInput](/core/text-input/), [NativeSelect](/core/native-select/), [Textarea](/core/textarea/), etc.)
under the hood. You _do not need to wrap your inputs with it, as it is already included in all of them_.
Use `InputWrapper` only when you want to create custom inputs.

## inputWrapperOrder

`inputWrapperOrder` allows configuring the order of `InputWrapper` parts.
It accepts an array of four elements: `label`, `input`, `error` and `description`.
Note that it is not required to include all of them – you can use only those that you need.
Parts that are not included will not be rendered.

## required and withAsterisk props

All components that are based on `InputWrapper` support `required` and `withAsterisk` props.
When set to true, both of these props will add a red asterisk to the end of the label.
The only difference is whether the input element will have the `required` attribute. Example with
the [TextInput](/core/text-input/) component:

```vue
<script setup lang="ts">
import { TextInput } from '@mantine-vue/core'
</script>

<template>
  <!-- Will display required asterisk and add `required` attribute to the input element -->
  <TextInput label="test-label" required />

  <!-- Will only display the asterisk, `required` attribute is not added to the input element -->
  <TextInput label="test-label" withAsterisk />
</template>
```

## error prop

All inputs that use `InputWrapper` under the hood support the `error` prop.
When set to `true`, it will add a red border to the input. You can also pass a string to display
an error message below the input. To only display an error message without a red border, set the `error` prop
to a string and `:withErrorStyles="false"`:

Default props on `Input` and `InputWrapper` cascade to every component that is built on top of
them. This is the easiest way to apply the same `size`, `radius`, `variant`, `withAsterisk` or any
other shared prop to all inputs at once. Per-component default props always win over the shared ones:

## Accessibility

If you use `Input` component without associated label element, set `aria-label`:

```vue
<script setup lang="ts">
import { Input } from '@mantine-vue/core'
</script>

<template>
  <!-- Accessible – labelled by aria-label -->
  <Input aria-label="Your email" />

  <!-- Accessible – labelled by the label element -->
  <label for="my-email">Your email</label>
  <Input id="my-email" />
</template>
```

When you use `Input` with `InputWrapper` it is required to set `id` on both components
to connect label and other elements with the input:

```vue
<script setup lang="ts">
import { useId } from '@mantine-vue/hooks'
import { Input, InputWrapper } from '@mantine-vue/core'

const id = useId()
</script>

<template>
  <InputWrapper label="Your email" :id="id">
    <Input :id="id" />
  </InputWrapper>
</template>
```
