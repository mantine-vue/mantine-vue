## Controlled

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { Chip } from '@mantine-vue/core'

const checked = ref(false)
</script>

<template>
  <Chip :checked="checked" @change="checked = !checked">My chip</Chip>
</template>
```

## Chip.Group

The `Chip.Group` component manages the state of child `Chip` components.
Set the `multiple` prop to allow multiple chips to be selected at a time:

<Demo data={ChipDemos.group} />

## Controlled Chip.Group

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { Chip, ChipGroup } from '@mantine-vue/core'

// string value when multiple is false (default)
const single = ref('react')

// array of strings value when multiple is true
const multiple = ref(['react'])
</script>

<template>
  <!-- Single selection -->
  <ChipGroup :multiple="false" :value="single" @change="single = $event">
    <Chip value="react">React</Chip>
    <Chip value="ng">Angular</Chip>
    <Chip value="svelte">Svelte</Chip>
    <Chip value="vue">Vue</Chip>
  </ChipGroup>

  <!-- Multiple selection -->
  <ChipGroup multiple :value="multiple" @change="multiple = $event">
    <Chip value="react">React</Chip>
    <Chip value="ng">Angular</Chip>
    <Chip value="svelte">Svelte</Chip>
    <Chip value="vue">Vue</Chip>
  </ChipGroup>
</template>
```

## Accessibility

`Chip` and `Chip.Group` components are accessible by default – they are built with native
radio/checkbox inputs, so all keyboard events work the same as with native controls.
