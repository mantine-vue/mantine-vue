## Tooltip children

Tooltip requires an element or a component as a single child –
strings, fragments, numbers and multiple elements/components are not supported.
Custom components must expose a root element ref; all Mantine components support ref out of the box.

```vue
<template>
  <!-- OK – native element -->
  <Tooltip label="OK">
    <button>Native button – ok</button>
  </Tooltip>

  <!-- OK – Mantine component -->
  <Tooltip label="OK">
    <Badge>Mantine component – ok</Badge>
  </Tooltip>
</template>
```

## Change events

Events that trigger the tooltip can be changed with the `events` prop:

```vue
<template>
  <Tooltip label="Tooltip" :events="{ hover: true, focus: true, touch: false }">
    <button>target</button>
  </Tooltip>
</template>
```

## Change transition

Tooltip is built with the [Transition](/core/transition/) component; use `transitionProps` to customize it:

```vue
<template>
  <Tooltip
    label="Tooltip with custom transition"
    :transition-props="{ transition: 'skew-up', duration: 300 }"
  >
    <Button>Button with tooltip</Button>
  </Tooltip>
</template>
```

## Accessibility

Tooltip follows [WAI-ARIA recommendations](https://www.w3.org/TR/wai-aria-practices/#tooltip):

- The tooltip body has `role="tooltip"` attribute
- The target element has `aria-describedby` attribute
- `Tooltip.Floating` is ignored by screen readers

By default, Tooltip is not triggered by focus events. Set the `events` prop to enable
focus/blur tooltip events for screen reader support:

```vue
<template>
  <Tooltip label="Tooltip" :events="{ hover: true, focus: true, touch: false }">
    <Button>Button with tooltip</Button>
  </Tooltip>
</template>
```
