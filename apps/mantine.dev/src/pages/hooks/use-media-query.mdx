## Usage

The `use-media-query` hook subscribes to media queries.
It receives a media query as an argument and returns a `Readonly<Ref<boolean>>` that is `true` when the given media query matches the current state.
The hook relies on the `window.matchMedia()` [API](https://developer.mozilla.org/en-US/docs/Web/API/Window/matchMedia)
and will return `false` if the API is not available, unless an initial value is provided in the second argument.

## Reactive query

The `query` argument accepts a `MaybeRefOrGetter<string>`, so it can be a plain string, a
`Ref<string>`, or a getter function. When the query value changes, the hook detaches the previous
`matchMedia` listener and attaches a new one for the updated query:

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { useMediaQuery } from '@mantine-vue/hooks'

const query = ref('(min-width: 40em)')
const matches = useMediaQuery(query)
</script>
```

## Server-Side Rendering

During server-side rendering, the hook will always return `false` as the `window.matchMedia` API is not available.
If that is not the desired behavior, you can override the initial value:

```ts
import { useMediaQuery } from '@mantine-vue/hooks'

// Set the initial value in the second argument and getInitialValueInEffect option to false
const matches = useMediaQuery('(max-width: 40em)', true, {
  getInitialValueInEffect: false,
})
```

## Definition

```ts
interface UseMediaQueryOptions {
  getInitialValueInEffect?: boolean
}

function useMediaQuery(
  query: MaybeRefOrGetter<string>,
  initialValue?: boolean,
  options?: UseMediaQueryOptions,
): Readonly<Ref<boolean>>
```

## Exported types

`UseMediaQueryOptions` type is exported from `@mantine-vue/hooks` package,
you can import it in your application:

```ts
import type { UseMediaQueryOptions } from '@mantine-vue/hooks'
```
