## Usage

The `use-debounced-state` hook debounces value changes.
This can be useful when you want to perform a heavy operation based on a ref value,
for example, sending a search request. Unlike [use-debounced-value](/hooks/use-debounced-value/), it
is designed to work with uncontrolled inputs.

<Demo data={UseDebouncedStateDemos.usage} />

## Differences from use-debounced-value

- You do not have direct access to the non-debounced value.
- It is designed for uncontrolled inputs: the bound `value` only re-renders once the debounce
  timer resolves, so it does not update on every keystroke like a fully controlled input.
- It does not work with custom state providers or props, it always uses an internal `ref`.

## Definition

```ts
interface UseDebouncedStateOptions {
  leading?: boolean
}

type UseDebouncedStateReturnValue<T> = [Ref<T>, (nextValue: T | ((value: T) => T)) => void]

function useDebouncedState<T>(
  initialValue: T,
  wait: number,
  options?: UseDebouncedStateOptions,
): UseDebouncedStateReturnValue<T>
```

## Exported types

The `UseDebouncedStateOptions` and `UseDebouncedStateReturnValue` types are exported from the `@mantine-vue/hooks` package;
you can import them in your application:

```ts
import type { UseDebouncedStateOptions, UseDebouncedStateReturnValue } from '@mantine-vue/hooks'
```
