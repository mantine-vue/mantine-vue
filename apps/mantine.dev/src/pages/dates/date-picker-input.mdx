## DatePicker props

`DatePickerInput` supports most of the [DatePicker](/dates/date-picker) props.
Read through the [DatePicker](/dates/date-picker) documentation to learn about all component features that are not listed on this page.

## Usage

<Demo data={DatePickerInputDemos.usage} />

## Multiple dates

Set `type="multiple"` to allow users to pick multiple dates:

<Demo data={DatePickerInputDemos.multiple} />

## Dates range

Set `type="range"` to allow users to pick a date range:

<Demo data={DatePickerInputDemos.range} />

## Presets

Use the `presets` prop to add custom date presets. Presets are displayed next to the calendar:

<Demo data={DatePickerInputDemos.presets} />

## Open picker in modal

By default, [DatePicker](/dates/date-picker) is rendered inside [Popover](/core/popover).
You can change that to [Modal](/core/modal) by setting `dropdownType="modal"`:

<Demo data={DatePickerInputDemos.modal} />

## Value format

Use the `valueFormat` prop to change the [dayjs format](https://day.js.org/docs/en/display/format) of the value label:

<Demo data={DatePickerInputDemos.valueFormat} />

## Clearable

Set the `clearable` prop to display a clear button in the right section. Note that if you set the `rightSection`
prop, the clear button will not be displayed.

<Demo data={DatePickerInputDemos.clearable} />

## Min and max dates

`minDate` and `maxDate` props define the minimum and maximum dates that can be picked:

<Demo data={DatePickerInputDemos.minMax} />

## Input props

`DatePickerInput` is built on top of [InputBase](/core/input), so it supports the same props: `label`,
`description`, `error`, `size`, `variant`, `leftSection`, `rightSectionWidth`, `required`, `disabled`, etc.
