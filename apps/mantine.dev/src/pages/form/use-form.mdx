## Usage

`useForm` creates a form object with reactive values, validation state and input helpers.

<Demo data={FormDemos.usage} />

## API overview

```ts
import { useForm } from '@mantine-vue/form'

const form = useForm({
  initialValues: {
    email: '',
    accepted: false,
    user: {
      firstName: 'John',
      lastName: 'Doe',
    },
  },
})
```

### Submit

`form.onSubmit` validates values before calling your submit handler. By default, it prevents default browser submit behavior.

```vue
<script setup lang="ts">
import { useForm } from '@mantine-vue/form'

const form = useForm({
  initialValues: { email: '' },
})

const handleSubmit = form.onSubmit(
  (values, event) => {
    console.log(values, event)
  },
  (errors, values, event) => {
    console.log(errors, values, event)
  },
)
</script>

<template>
  <form @submit="handleSubmit" />
</template>
```

Use `onSubmitPreventDefault` to control when `event.preventDefault()` is called:

```ts
useForm({
  onSubmitPreventDefault: 'always', // default
})

useForm({
  onSubmitPreventDefault: 'never',
})

useForm({
  onSubmitPreventDefault: 'validation-failed',
})
```

Use `form.onReset` to reset form values and status from a form reset event:

```vue
<script setup lang="ts">
const handleSubmit = form.onSubmit((values) => {
  console.log(values)
})
</script>

<template>
  <form @submit="handleSubmit" @reset="form.onReset" />
</template>
```

If you call `form.onSubmit` inline in a Vue template, call the returned handler with `$event`:

```vue
<form @submit="form.onSubmit((values) => console.log(values))($event)" />
```

## Controlled and uncontrolled modes

The default mode is `controlled`. In `uncontrolled` mode, `form.key(path)` changes when the field value is updated, which lets you force input remounts when needed.

```vue
<TextInput :key="form.key('email')" v-bind="form.getInputProps('email')" />
```

## Watch fields

Use `watch` to subscribe to field value changes. It returns a cleanup function.

```ts
const cleanup = form.watch('email', ({ previousValue, value, touched, dirty }) => {
  console.log(previousValue, value, touched, dirty)
})

cleanup()
```

Set `cascadeUpdates: true` to notify nested watchers when parent paths change and parent watchers when nested paths change.
